# 도메인 및 검색 노출 운영 가이드

## 목적
- `https://miokbook.vercel.app` 를 임시 주소가 아니라 실제 서비스 주소처럼 쓰지 않도록 정리한다.
- 검색엔진이 `미옥서원` 사이트를 안정적으로 인덱싱하도록 운영 기준을 문서화한다.

## 결론
- `vercel.app` 주소도 검색은 될 수 있다.
- 하지만 실서비스 검색 노출은 `커스텀 도메인` 연결을 전제로 운영하는 것이 맞다.
- 권장 예시:
  - `https://miokbook.com`
  - `https://www.miokbook.com`

## 꼭 해야 하는 것
1. 도메인 구매
- 가비아, 후이즈, Cloudflare Registrar 등에서 구매

2. Vercel 에 커스텀 도메인 연결
- Vercel Project > Settings > Domains
- `miokbook.com`
- 필요하면 `www.miokbook.com`도 같이 연결

3. 환경변수 고정
- Vercel Production 환경변수에 아래 값 고정
- `NEXT_PUBLIC_SITE_URL=https://miokbook.com`

이 값이 중요한 이유:
- canonical URL 기준이 된다.
- robots 의 sitemap 경로 기준이 된다.
- OG 공유 주소 기준이 된다.

4. 대표 도메인 하나로 통일
- `miokbook.com` 을 대표로 쓸지
- `www.miokbook.com` 을 대표로 쓸지
- 하나만 canonical host 로 고정한다.

## 권장 운영 방식
- 대표 주소: `https://miokbook.com`
- `www.miokbook.com` 은 `miokbook.com` 으로 리다이렉트
- `*.vercel.app` 직접 접근도 대표 주소로 리다이렉트

현재 코드상:
- 프로덕션에서는 host 가 canonical 과 다르면 308 redirect 되도록 되어 있다.
- 따라서 `NEXT_PUBLIC_SITE_URL` 또는 `VERCEL_PROJECT_PRODUCTION_URL` 설정이 중요하다.

## 검색 노출 체크리스트
1. Google Search Console 등록
- 도메인 속성 또는 URL prefix 속성 등록
- 권장: 도메인 속성 등록

2. 사이트맵 제출
- 제출 주소:
  - `https://miokbook.com/sitemap.xml`

3. robots 확인
- 확인 주소:
  - `https://miokbook.com/robots.txt`

4. 대표 페이지 수동 색인 요청
- `/`
- `/books`
- `/new-books`
- `/bestsellers`
- 주요 도서 상세 몇 개
- `/content`, `/notices`

## 운영 중 확인할 것
- 배포 후 페이지 소스에서 canonical 이 커스텀 도메인으로 잡히는지
- sitemap 의 각 URL 이 `vercel.app` 이 아니라 커스텀 도메인으로 생성되는지
- Search Console 에서 중복 URL, 리디렉션 오류, 크롤링 실패가 없는지

## `vercel.app` 주소를 그대로 쓰면 생길 수 있는 문제
- 브랜드 신뢰도 저하
- 공유 URL 일관성 저하
- canonical 혼선
- 추후 도메인 전환 시 검색엔진 재학습 비용 증가

## 홈 신간 섹션 관련 메모
- 홈 신간 노출은 별도 임시 경로가 아니라 공용 신간 목록 기준으로 맞춰져 있어야 한다.
- 프로덕션에서는 ISR / cache 영향이 있으므로, 도메인 연결 후에도 캐시 반영 시간이 잠깐 존재할 수 있다.
