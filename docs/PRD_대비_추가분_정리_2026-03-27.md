# PRD 대비 추가분 정리

기준일: 2026-03-27

기준 문서
- 최신 PRD 기준: 루트 `PRD FINAL` 문서, `docs/수정prd.md`
- 현재 진행/완료 기준: `docs/PROGRESS.md`

이 문서는 현재까지 실제로 반영된 작업과, 최신 PRD 기준으로 아직 추가 작업으로 봐야 하는 부분을 분리해서 정리한 문서다.

---

## 1. 현재 완료로 볼 수 있는 내역

아래 항목은 최근 작업 내역과 현재 코드 기준으로 이미 반영됐거나, 최소한 운영 가능한 수준까지 연결된 것으로 본다.

### 1-1. 스토어 콘텐츠/배너 영역
- 메인 상단 캐러셀 배너별 링크 연동
- 캐러셀 배너 링크를 상세 슬러그 경로로 직접 입력 가능한 구조로 변경
- `/content` 목록 최신순 정렬
- `/content` 상단 쇼케이스와 하단 전체 영상 UI 정리
- 영상 상세 `자세히 보기` 페이지 상단 2열 레이아웃 적용
- 영상 상세 우측 포스터용 `관련 이미지` 입력 및 노출 연결
- 외부 재생 주소가 직접 mp4 링크일 때 인라인 재생 분기 보강

### 1-2. 관리자 CMS/운영 화면
- 마케팅 배너/팝업 관리 화면 유지
- 유튜브 콘텐츠 등록/수정 화면 유지
- 유튜브 콘텐츠에 `관련 이미지` 입력 항목 추가
- 대량주문, 주문, 이벤트, 콘텐츠, 도서 관리 화면 기본 운영 구조 유지

### 1-3. 전자서명/계약서 흐름
- 유캔싸인 웹훅 라우트 추가
- 유캔싸인 API 키 환경변수 연결 구조 정리
- 계약서 서명 요청/임베딩/웹훅 반영 흐름 연결
- 템플릿 기반 전자서명 연동 문서 정리

### 1-4. 운영 중 발견된 오류 보정
- 견적문의 모달 `undefined.length` 오류 수정
- 영상 카드 링크 구조 보정
- `/content` 상단/하단 `자세히 보기` 스타일 정리
- 슬러그 중복 데이터가 있을 때 최신 행 우선 조회하도록 보정

## 2. PRD와 비교했을 때 추가분으로 남는 내역

아래 항목은 PRD에는 명시되어 있지만, 현재 상태를 PRD 완료로 보기에는 아직 검증이나 마감 정리가 더 필요한 부분이다.

## [추가분]

### 2-1. 팝업 스토어 노출 최종 검증
- PRD 기준으로는 팝업이 관리자 저장 이후 실제 스토어에 노출되고, 닫기/오늘 하루 보지 않기까지 운영 검증이 끝나야 완료다.
- 현재는 기능 구조가 있어도 운영 검증 결과를 문서화한 상태로 보긴 어렵다.

추가 필요:
- 스토어 실제 노출 확인
- 오늘 하루 보지 않기 동작 확인
- 모바일 표시 확인

### 2-2. `main_top`, `sidebar` 배너 운영 검증
- PRD는 배너 위치를 `main_hero`, `main_top`, `sidebar`로 나눠 본다.
- 최근 캐러셀 링크와 메인 히어로 영역은 많이 보강됐지만, 각 위치별 운영 검증이 PRD 완료 수준으로 정리되진 않았다.

추가 필요:
- 위치별 등록값 확인
- 위치별 실제 노출 확인
- 클릭 링크 이동 확인

### 2-3. 업로드 안정화 최종 체크리스트
- 배너, 팝업, 콘텐츠 포스터 업로드는 실제 운영에서 중요하다.
- 과거 업로드 오류 이슈가 있었던 만큼, PRD 완료 기준으로는 업로드 성공, 저장 URL, 공개 페이지 노출까지 체크리스트가 필요하다.

추가 필요:
- 배너 이미지 업로드 확인
- 팝업 이미지 업로드 확인
- 콘텐츠 포스터 업로드 확인
- 저장 후 공개 페이지 노출 확인

### 2-4. 하이브리드 검색 고도화
- PRD는 내부 DB miss 시 외부 fallback, ISBN dedup, 리드 수집까지 포함한다.
- 현재 검색은 많이 보강됐지만 PRD 기준의 검색 운영 문서와 최종 검증은 남아 있다.

추가 필요:
- 내부 결과 없음 상황의 fallback 검증
- ISBN dedup 검증
- 리드 수집 여부 정책 결정

### 2-5. 주문 CSV/Excel 운영 포맷 정리
- PRD는 운영자가 실제로 바로 사용할 수 있는 주문 export를 요구한다.
- 현재 기능이 있어도 PRD 완료로 보려면 컬럼, 포맷, 운영 사용 흐름이 정리돼 있어야 한다.

추가 필요:
- export 컬럼 정의
- 배송/연락처/상품내역 출력 형식 점검
- 운영 전달용 포맷 확정

### 2-6. 취소/반품/재고 복원 운영 검증
- PRD는 주문 상태와 재고 복원이 정확히 맞물리는 것을 중요하게 본다.
- 현재 기본 흐름은 있으나, 취소/반품 완료 시 재고와 상태가 운영 기준으로 완전히 일치하는지 최종 검증이 더 필요하다.

추가 필요:
- 취소 시 재고 복원 확인
- 반품 완료 시 상태값 확인
- 상태 이력/운영 처리 기준 확인

### 2-7. 이벤트 신청/취소 원자성 검증
- PRD는 이벤트 정원 초과 방지와 취소 시 카운트 복원이 중요하다.
- 현재 로직이 있어도 동시성/취소까지 포함한 운영 검증 기록은 남겨야 한다.

추가 필요:
- 정원 초과 방지 확인
- 취소 시 카운트 감소 확인
- 관리자 참가자 목록과 일치 확인

### 2-8. 리뷰 집계 최적화 검증
- PRD는 리뷰 집계를 O(1) 구조로 유지하는 것을 요구한다.
- 코드상 보강이 있어도 실제 반영값이 목록/상세/정렬에 일관되게 반영되는지 체크가 남아 있다.

추가 필요:
- 리뷰 생성 후 평점/개수 반영 확인
- 상세/목록 정렬 일치 확인

### 2-9. Product JSON-LD / SEO 구조화 데이터
- PRD는 도서 상세 페이지의 구조화 데이터를 명시한다.
- 일반 메타는 있어도 Product JSON-LD가 PRD 기준으로 완성됐는지 별도 확인이 필요하다.

추가 필요:
- 도서 상세 JSON-LD 적용 여부 확인
- Rich Results 기준 검증

### 2-10. GA4 / Analytics 이벤트 카탈로그
- PRD는 검색, 장바구니, 결제, 이벤트 신청 같은 운영 이벤트 추적까지 포함한다.
- 현재 연결 여부와 별개로, 운영 문서 관점에서 이벤트 목록과 측정 기준 정리가 더 필요하다.

추가 필요:
- 추적 대상 이벤트 목록 정리
- 이벤트별 파라미터 정리
- 운영 확인 방법 정리

### 2-11. 콘텐츠/스토리형 페이지 운영 규칙
- PRD는 콘텐츠와 큐레이션을 단순 CRUD가 아니라 운영형 구조로 본다.
- 현재 콘텐츠 기능은 많이 들어가 있지만, 본문 입력 규칙과 공개 렌더링 기준을 운영 문서로 정리할 필요가 있다.

추가 필요:
- 본문 입력 규칙
- 이미지 삽입 규칙
- 공개 렌더링 기준

### 2-12. 운영 가이드 문서화
- 최근 작업으로 기능은 늘었지만, 운영자 관점에서 “어디에 무엇을 넣으면 어디에 노출되는지”를 한 번에 보는 문서가 더 필요하다.

추가 필요:
- 메인 캐러셀 배너 입력 가이드
- 콘텐츠 포스터 입력 가이드
- 외부 영상 링크 입력 가이드
- Firebase Storage 업로드 가이드

---

## 3. 우선순위 제안

현재 기준으로는 아래 순서가 맞다.

1. 배너/팝업/업로드처럼 실제 화면 노출에 바로 영향 주는 운영 검증
2. 주문/반품/재고/이벤트처럼 상태 무결성이 중요한 항목 검증
3. 검색 fallback, SEO, Analytics 같은 고도화 항목 정리
4. 운영 가이드 문서 정리

---

## 4. 메모

이번 비교에서 중요한 점은, 기능이 “아예 없는 것”보다 “기능은 있지만 PRD 완료로 보기엔 운영 검증과 문서화가 덜 된 것”이 더 많다는 점이다.

즉 현재 상태는:
- 주요 기능과 화면은 상당 부분 반영됨
- 하지만 PRD 완료 기준으로 보려면 운영 검증, 체크리스트, 가이드 문서가 추가로 필요함
