# 유캔싸인(YouCanSign) 전자서명 연동 계획

## 개요

대량구매 서비스의 도서 납품 계약서 체결에 유캔싸인 전자서명을 연동한다.
현재는 버튼 클릭 시 DB에 `signed: true`만 저장하는 가짜 서명 구조이므로, 실제 법적 효력을 가진 전자서명으로 교체한다.

---

## 연동 전후 흐름 비교

### 현재 (가짜 서명)
```
고객 견적 요청
  → 어드민이 견적서 작성 (admin/bulk-orders/[id])
  → 계약서 링크 고객에게 수동 공유 (/bulk-order/contract/[id])
  → 고객이 도장 영역 클릭
  → DB contract.signedByEul = true, status = 'contracted' 저장
```

### 연동 후 (유캔싸인)
```
고객 견적 요청
  → 어드민이 견적서 작성 후 저장
  → "전자서명 요청" 버튼 클릭
  → 서명 방식 선택 다이얼로그
       ├─ [임베딩] → 유캔싸인 API로 sign request 생성
       │             → embed URL 발급
       │             → 계약서 페이지 내 iframe으로 서명 진행
       └─ [카카오 발송] → 유캔싸인 API로 sign request 생성
                         → 고객 핸드폰 번호로 카카오톡 알림 발송
                         → 고객이 링크 클릭 후 서명 진행
  → 서명 완료 시 유캔싸인 Webhook 수신
  → bulk_orders.status = 'contracted' 자동 업데이트
```

## 작업 시작 전 필요한 것

`app.ucansign.com/developer` 로그인 후 아래 내용 확인 및 수집:

- [ ] **API Key** 발급 → Vercel 환경변수 `UCANSIGN_API_KEY`로 설정
- [ ] **서명 요청 생성 API** 엔드포인트 + 요청/응답 파라미터 예시
- [ ] **임베딩 URL 발급** 방법 (서명자 토큰 방식 여부)
- [ ] **Webhook** 설정 방법 및 이벤트 스펙 (서명 완료 이벤트)
- [ ] 카카오톡 발송 시 수신자 정보 파라미터 (이름, 전화번호)

## 구현할 파일 목록

### 신규 생성

| 파일 | 역할 |
|---|---|
| `src/lib/ucansign.ts` | 유캔싸인 API 클라이언트 (sign request 생성, embed URL 발급) |
| `src/app/api/admin/bulk-orders/[id]/sign-request/route.ts` | 어드민: 서명 요청 생성 API (임베딩 / 카카오 선택) |
| `src/app/api/webhooks/ucansign/route.ts` | 유캔싸인 Webhook 수신 → 계약 완료 처리 |

### 수정

| 파일 | 변경 내용 |
|---|---|
| `src/app/(admin)/admin/bulk-orders/[id]/page.tsx` | 견적서 저장 후 "전자서명 요청" 버튼 + 방식 선택 다이얼로그 추가 |
| `src/app/(store)/bulk-order/contract/[id]/page.tsx` | 임베딩 방식 선택 시 iframe으로 유캔싸인 서명 화면 표시 |
| `src/app/api/admin/bulk-orders/[id]/route.ts` | sign_request_id, sign_method 필드 저장 지원 |

## 환경변수 (Vercel에 추가 필요)

```env
UCANSIGN_API_KEY=              # 유캔싸인 개발자 센터에서 발급
UCANSIGN_WEBHOOK_SECRET=       # Webhook 검증용 시크릿 (있을 경우)
```

## DB 변경 (bulk_orders 테이블)

현재 `contract` JSONB 컬럼에 아래 필드 추가:

```json
{
  "signedByEul": true,
  "signedAtEul": "2026-03-24T12:00:00Z",
  "eulName": "홍길동",
  "signMethod": "embedding",      // 추가: "embedding" | "kakao"
  "ucansignRequestId": "req_xxx", // 추가: 유캔싸인 서명 요청 ID
  "ucansignDocumentId": "doc_xxx" // 추가: 유캔싸인 문서 ID
}
```

## 구현 순서

1. **유캔싸인 API 문서 수집** (선행 필수)
2. `src/lib/ucansign.ts` API 클라이언트 작성
3. `sign-request/route.ts` 어드민 서명 요청 API 작성
4. `webhooks/ucansign/route.ts` Webhook 수신 처리 작성
5. 어드민 페이지 UI — "전자서명 요청" 버튼 + 방식 선택 모달
6. 고객 계약서 페이지 — 임베딩 iframe 표시
7. Vercel Webhook URL 등록 및 엔드투엔드 테스트
